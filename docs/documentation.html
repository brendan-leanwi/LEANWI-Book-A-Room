LEANWI-Book-A-Room Documentation

Welcome to the LEANWI-Book-A-Room plugin documentation.
This guide will help you configure and use the plugin efficiently.

Features Include

  • All functionality is directly put onto your website's pages, making it highly customizable and versatile.
  • Community members book meeting rooms based on your library's use policies.
  • Once a time slot is booked, the room is immediately blocked for all other users.
  • Multiple ways for community members and staff to view room availability.
  • Multiple ways for staff to search for and adjust bookings.
  • Staff can add recurring bookings easily for events like Storytime and make adjustments when needed.
  • Ability to book appointments with staff members for services like tech help or notary appointments.
  • Staff can easily track and cancel erroneous bookings.
  • An admin dashboard allows full control of all staff and user settings.
  • Ability to send a feedback request after a booking.
  • Fully configurable audiences and categories for reporting with flexible filters and CSV file export.
  • The ability to add pricing to any room or staff appointment and track payments.
  • Supports reCAPTCHA for security.

Quick Product Overview

The LEANWI-Book-A-Room plugin is set up in two parts:

  1. The admin settings and venue details are configured under the LEANWI-Book-A-Room menu.
  2. Shortcodes are added to your pages.

Typical Pages:

Rooms Landing Page (Rooms Example Page)

This is the main page for users to browse venues, learn how to book, and view policies.

  • Users can view availability for specific rooms.
  • Users can search for available rooms based on a desired date and time.
  • Users can search for an existing booking using their Booking ID. (Shortcode: [booking_search])

Venue Pages (Venue Example Page)

Venues are configured in the plugin settings, but each venue needs a page with the corresponding shortcode:

Example: [venue_details venue_id="1"] (for venue ID 1).

Recurring Bookings Page (Recurring Bookings Example Page)

Staff can create recurring bookings by adding the shortcode [staff_recurring_bookings] to a page.

Payments and Feedback Monitoring Page (Optional) (Payments and Feedback Example Page)

To track payments and request feedback, add the shortcode [staff_payment_search] to a page.

Check Availability Page (Room Availability Example Page)

This page allows users and staff to check venue availability for a selected day using the [venue_grid] shortcode.

Staff can also search for bookings by name or organization using [staff_name_search].
